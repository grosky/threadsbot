### Threads tester invite — баг
Новые юзеры не получают приглашение через Threads даже после добавления в Meta dashboard. Причины:
- Возможно неправильный username (с `@`)
- Старая версия Threads-приложения у юзера
- Приглашение приходит на email привязанный к Instagram

Добавить в бот:
- Подсказку «Если не видишь приглашение → открой threads.com/settings на десктопе»
- Команду `/invite_help` с пошаговой инструкцией

**Долгосрочное решение:** App Review.

### UTM-генератор для админа
- Команда `/utm` → бот спрашивает `source`
- Возвращает ссылку `t.me/aithreadbot?start=ref_<admin_id>_<source>`
- Сохраняем `source` в `referrals` таблице
- `/stats <source>` → «N кликов, M активаций, K оплат»

### Инструкции в боте
- Записать скринкаст или 5-7 скринов:
  - /start → Welcome
  - Free trial → онбординг
  - Меню → Создание → Сгенерить
  - Получение поста + post-actions
  - Подключение Threads (OAuth)
  - Опубликованный пост
- Команда `/help` показывает их юзеру

## 🛠 Технический долг

- bot.db живёт в `/data` (Railway Volume) — backup стратегия не настроена
- Логи Railway хранятся 7 дней — для аудита нужен внешний sink (LogTail/Datadog)
- Token refresh для Threads — long-lived живёт 60 дней, нужен фоновый job для refresh
